Areas de GIT

Working Directory

Es el area de trabajo, donde rara vez vamos a estar utilizando la linea de comandos, salvo por el inicio del proyecto o a lo sumo la eliminacion del mismo. Se refiere a la carpeta donde iniciamos el repositorio y basicamente donde vamos a crear, modificar y eliminar los archivos que formen parte de nuestro proyecto.

  • dir || ls muestra el contenido de la carpeta actual
  • mkdir <name> crea una carpeta con el nombre indicado
  • cd <path> abre el directorio especificado
  • git init inicializa un repositorio de git
  • rm -rf .git elimina el repositorio de manera forzada

Staging Area (INDEX)

El area de control de Cambios, corresponde a un archivo que se encuentra en la carpeta .git, y se utiliza para realizar las capturas de pantalla del codigo (snapshot) de los diferentes archivos del proyecto. Aqui se realiza el seguimiento de los mismos, para realizar la confirmacion y poder acceder a los diferentes cambios dentro de la historia de nuestro proyecto. Es importante tener en cuenta que todo archivo que no sea agregado al INDEX, no se podra recuperar en caso de ser eliminado.

  • git status muestra el estado de los archivos con respecto a la ultima confirmacion, en caso que exista.
    • untracked: al archivo no se le esta realizando seguimiento.
    • staged: los cambios fueron agregados, pero aun no estan confirmados
    • modified: el archivo difiere con respecto a lo almacenado en la ultima confirmacion
    • commited los cambios fueron almacenados en una confirmacion
  • git add <file> agrega al INDEX el/los archivo/s seleccionados
  • git rm --cached <file> quita del INDEX el archivo seleccionado.
  • git restore --staged <file> remueve los cambios de los archivos agregados al INDEX

Repository

Area donde se almacenan los cambios, mediante confirmaciones, en formato BLOB(Binary Large Object File). Solamente se confirman las modificaciones agregadas al INDEX; y mediante apuntadores podemos navegar dentro del historial.

  • git commit abre el editor para realizar la confirmacion
  • git blame <file> muestra linea por linea los autores de las confirmaciones, en caso que existan
  • git log muestra el historial de confirmaciones en orden descendente
  • git remote add <alias> <url> agrega una direccion remota, asignandole un alias
  • git fetch <remote> obtiene el historial de cambios de un repositorio remoto
  • git push <remote> <url> envia a un repositorio remoto los cambios
